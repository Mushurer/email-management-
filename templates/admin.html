External API Integration

External systems can integrate with the DMS using these API endpoints:

Single Student Check
POST /api/external/check-student 
{
  "reg_number": "R123456A",
  "domain": "university.edu"
}

Response:

{
  "exists": true,
  "status": "active", // active, blacklisted, exonerated, not_found
  "reg_number": "R123456A",
  "total_cases": 2,
  "requesting_domain": "university.edu",
  "timestamp": "2024-01-01T12:00:00"
}
Batch Student Check
POST /api/external/batch-check 
{
  "reg_numbers": ["R123456A", "R789012B"],
  "domain": "university.edu"
}
Domain Verification
POST /api/external/verify-domain 
{
  "domain": "university.edu"
}
Integration Notes:
  • All requests must include Content-Type: application/json
  • Educational domains (.edu, .ac., university., college.) are pre-authorized
  • Batch requests are limited to 100 registration numbers
  • All API calls are logged for audit purposes
  • Rate limiting may apply for high-volume usage
